# 认知库

记**想通的道理**，不是记决策、也不是记事实。

- `docs/` 记拍板的决策（已经定了的）
- **认知库记决策的"上游"**：为什么这么定、学到了什么、踩坑想通了什么
- 记忆库（`~/.claude/.../memory/`）记原子事实，给 Claude 当上下文

## 怎么用

1. **随手记** → 任何时刻冒出来的想法，一行甩进 [INBOX](INBOX.md)，不分类、不讲究，先抓住。
2. **定期提炼** → 跟 Claude 说一声"整理认知库"，它把 INBOX 的毛坯归类、补背景、连关联、标状态，归到下面四个分区。
3. **三个出口**（成熟的认知不要烂在这里）：
   - 🌳 喂给**记忆库** → 变成 Claude 的长期上下文，影响它怎么帮你
   - 🌳 改写成 **journey 建设日志** → build-in-public 的好料
   - 当一条认知足够指导一个具体选择 → 升级成 **`docs/` 决策**

## 四个分区

| 文件 | 装什么 |
|---|---|
| [技术工程](技术工程.md) | AI / agent / 全栈架构的理解和心智模型 |
| [产品商业](产品商业.md) | 做产品、找付费点、商业模式、用户 |
| [领域行业](领域行业.md) | 某个具体行业/场景的 know-how（如商家网页自动化） |
| [方法自我](方法自我.md) | 工作方式、学习方法、跟 Claude 协作的模式 |

## 每条认知的模板

```markdown
## 一句话结论（认知本身）
- 来由：什么事 / 什么时刻让你想通的
- 展开：为什么、边界在哪、反例是什么
- 关联：[[已有的某条认知 或 某个决策文档]]
- 状态：🌱raw / 🌿成型 / 🌳已沉淀
```

`[[关联]]` 很重要——认知的价值在**连成网**。孤立的金句记了也白记。

## 演化规则

早期一条认知 = 一个 `##` 小节，别搞一条一文件。等某条长大到能独立成篇，再拆出去单独建文件。保持轻，否则你不会来填。
